# 1. LLMOの定義と目的

---

## LLMOとは何か

**LLMO = Large Language Model Optimization**

ChatGPT、Claude、Gemini、Perplexityなどの大規模言語モデル（LLM）が、コンテンツを正確に理解・抽出・引用できるように最適化するプロセス

**目的**
AIが生成する回答の中で、自社が信頼できる情報源として引用・推奨されることを確保する

---

## なぜ今LLMOが必要なのか

### 市場データ（2024-2025年）

- **生成AIトラフィック**: 1,200%成長（2024年7月〜2025年2月）
- **ChatGPT月間ユーザー**: 6億人
- **Google Gemini月間ユーザー**: 3億5,000万人
- **Perplexity月間訪問者**: 9,080万人
- **AI検索優先ユーザー**: 10%（全インターネットユーザー）

**Gartner予測**: 2028年までに従来の検索トラフィックが50%減少

---

## SEOとLLMOの違い

| 側面 | 従来のSEO | LLMO |
|------|----------|------|
| **主な目標** | 検索結果での上位表示 | AI回答での引用・言及 |
| **対象** | Google、Bing | ChatGPT、Perplexity |
| **成功指標** | 順位、トラフィック | 引用頻度、言及精度 |
| **コンテンツ** | キーワード最適化 | 文脈関連性、明確さ |
| **クエリ** | 短いキーワード | 会話形式の質問 |

---

## SEOとLLMOの共通点

### 両者が重視する要素

✅ 正確で事実に基づいた情報  
✅ 包括的なトピックカバレッジ  
✅ 明確なヘッディング階層（H1〜H3）  
✅ Schema.orgマークアップ  
✅ 高品質な被リンク  
✅ E-E-A-T原則（経験、専門性、権威性、信頼性）

**重要**: Googleトップ3のサイトは、同じキーワードでAIツールに言及される確率が**最大77%**

# 2. 構造化データの活用

---

## なぜ構造化データが重要か

### 主な利点

- **AI引用率**: FAQスキーマで750%向上
- **CTR向上**: リッチリザルトで25-35%増加
- **AI可視性**: 15倍の改善
- **ナレッジグラフ**: ブランド権威の確立

**2025年**: GoogleのGeminiはAI Overviewsで構造化データを活用

---

## JSON-LDとは

**JSON-LD** = JavaScript Object Notation for Linked Data

### なぜJSON-LDを使うのか

✅ HTMLとのクリーンな分離
✅ 実装と更新が容易
✅ 大規模サイトに対応可能
✅ エラーが発生しにくい
✅ Googleの推奨形式第1位

---

## JSON-LDの基本構造

```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "@id": "https://example.com/article#id",
  "headline": "記事タイトル",
  "author": {
    "@type": "Person",
    "name": "著者名"
  }
}
```

---

## スキーマタイプの優先順位

| 優先度 | スキーマタイプ | 用途 | 影響 |
|--------|---------------|------|------|
| 1 | Organization | 企業情報 | エンティティ確立 |
| 2 | Person | 著者・チーム | E-E-A-T権威性 |
| 3 | Article | コンテンツ | AI引用 |
| 4 | FAQPage | Q&A | 音声検索 |
| 5 | Product | Eコマース | ショッピング結果 |

---

## Organizationスキーマ（必須）

```json
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "@id": "https://www.example.com/#organization",
  "name": "株式会社Example",
  "url": "https://www.example.com",
  "logo": {
    "@type": "ImageObject",
    "url": "https://www.example.com/logo.png"
  },
  "sameAs": [
    "https://www.facebook.com/example",
    "https://twitter.com/example"
  ]
}
```

**実装場所**: ホームページに配置

---

## FAQスキーマ（最重要）

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [{
    "@type": "Question",
    "name": "構造化データの実装方法は？",
    "acceptedAnswer": {
      "@type": "Answer",
      "text": "HTMLにJSON-LDスクリプトタグを追加します..."
    }
  }]
}
```

**効果**: AI引用率が750%向上

---

## 検証ツール

### 必須ツール（2024-2025年）

1. **Google Rich Results Test**
   - リッチスニペット適格性をテスト
   
2. **Schema Markup Validator**
   - Schema.org標準に対して検証
   
3. **Google Search Console**
   - デプロイ後の構造化データを監視

---

## 検証ワークフロー

1. **コードを書く** → テンプレート使用
2. **構文テスト** → JSON-LD Playground
3. **スキーマ検証** → Schema Markup Validator
4. **リッチリザルト確認** → Google Rich Results Test
5. **デプロイ** → ライブサイトに公開
6. **監視** → Search Console
7. **待機** → 検証に2〜14日間

---

## 一般的なエラーと修正

| エラー | 原因 | 修正 |
|--------|------|------|
| 必須プロパティの欠落 | 不完全なスキーマ | 全フィールド追加 |
| 無効な日付 | 間違った形式 | ISO 8601使用 |
| 不正なJSON | 構文エラー | 括弧・カンマ確認 |
| 画像が小さすぎる | <1200px | 最小1200px使用 |

# 3. FAQ・一次情報の設計

---

## なぜFAQがAIにとって重要か

- **AI優先**: LLMは自然なQ&A形式を好む
- **引用率**: FAQページは標準コンテンツより3〜5倍引用されやすい
- **構造**: 明確な意味信号を提供

---

## FAQ構造のベストプラクティス

### ❌ 不良構造

```
当社のサービス
コンサルティング、実装、サポートなど多くのサービスを提供。
詳細についてはお問い合わせください。
```

### ✅ AI最適化

```markdown
## どのようなサービスを提供していますか？

当社は3つの主要サービスを提供しています：
- 戦略コンサルティング（2〜4週間）
- 技術実装（4〜12週間）
- 継続的サポート（月額リテナー）
```

---

## FAQ作成ルール

### ✅ すべきこと

- 質問を見出し（H2/H3）として開始
- 最初の文で直接回答
- 35〜55語の回答を維持
- 自然言語の質問を使用
- ページごとに5〜10個のFAQ

### ❌ 避けるべきこと

- 曖昧な質問
- 回答を段落の深くに埋める
- 複数ページで同じFAQを複製

---

## 一次情報の構造化

### ❌ 不良定義

```
機械学習は、さまざまな業界で使用されている
重要な技術です。
```

### ✅ AI最適化定義

```markdown
## 機械学習とは何ですか？

機械学習は、明示的なプログラミングなしに
コンピューターがデータから学習できるようにする
人工知能の一種です。

**主な特徴**: データからパターンを学習、
時間とともに精度を向上
```

# 5. API・JSON形式での情報公開

---

## REST API設計原則

### 基本原則

- HTTPを標準プロトコルとして使用
- JSON形式をサポート（LLMが好む）
- 機械可読なドキュメント提供
- ステートレス通信

### RESTfulアーキテクチャ

- リソース（名詞）中心の設計
- 標準HTTPメソッド使用
- 例: `https://api.example.com/orders/123`

---

## HTTPメソッド

| メソッド | 目的 | 冪等 | 例 |
|---------|------|------|-----|
| GET | 取得 | はい | `GET /users/123` |
| POST | 作成 | いいえ | `POST /users` |
| PUT | 置換 | はい | `PUT /users/123` |
| PATCH | 更新 | いいえ | `PATCH /users/123` |
| DELETE | 削除 | はい | `DELETE /users/123` |

---

## robots.txtでAIクローラー制御

```txt
# OpenAI GPTBot (ChatGPT)
User-agent: GPTBot
Disallow: /

# Google AI (Gemini)
User-agent: Google-Extended
Disallow: /

# Perplexity AI
User-agent: PerplexityBot
Allow: /public/
Disallow: /

# 検索エンジンを許可
User-agent: Googlebot
Allow: /
```

---

## 新標準: llms.txt

**場所**: `https://example.com/llms.txt`

```markdown
# 会社名

## 概要
当社の事業説明

## 主要リソース
- /api/docs - APIドキュメント
- /blog - 技術ブログ
- /guides - ハウツーガイド

## API情報
ベースURL: https://api.example.com/v1
```
